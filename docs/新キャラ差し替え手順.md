# 新キャラ追加手順・注意点

このメモは、キャラクター追加時に起きやすい「隣キャラの混入」「髪の見切れ」「透明境界の荒れ」を防ぎつつ、Web版で複数キャラを切り替えられる形にするための手順です。

## キャラ別の置き場所

Web版は `src/character-config.js` に登録されたキャラを切り替えます。

現在の登録キャラ:

| キャラID | 表示名 | スライス画像の置き場所 |
|---|---|---|
| `upamune` | `upamune` | `public/characters/upamune/slices/` |
| `michiru_da` | `michiru_da` | `public/characters/michiru_da/slices/` |

追加・変更する場合は、キャラIDごとに以下の構成でスライス画像を置きます。

```text
public/characters/{キャラID}/slices/A/r0c0.webp ... r4c4.webp
public/characters/{キャラID}/slices/B/r0c0.webp ... r4c4.webp
...
public/characters/{キャラID}/slices/F/r0c0.webp ... r4c4.webp
```

1キャラあたり `6状態 × 25方向 = 150枚` です。

## 必要な素材

キャラごとに、以下の6枚を用意します。

| シート | 目 | 口 | ファイル名の例 |
|---|---|---|---|
| A | 開け | とじ | `A_目開け_口とじ.png` |
| B | 開け | 中間 | `B_目開け_口中間.png` |
| C | 開け | 開け | `C_目開け_口開け.png` |
| D | 閉じ | とじ | `D_目閉じ_口とじ.png` |
| E | 閉じ | 中間 | `E_目閉じ_口中間.png` |
| F | 閉じ | 開け | `F_目閉じ_口開け.png` |

条件:

- 4500×4500px の透過PNG
- 5×5グリッド、1コマ900×900px
- 行: 上向き → 下向き
  - `r0`: 上を見る
  - `r1`: やや上
  - `r2`: 正面
  - `r3`: やや下
  - `r4`: 下を見る
- 列: 左向き → 右向き
  - `c0`: 左を見る
  - `c2`: 正面
  - `c4`: 右を見る
- A〜Fでキャラ位置・角度・体の大きさは同じにする
- 目と口以外はできるだけ同一にする

## 重要な注意点

### 1. 900pxセルで単純に切らない

素材によっては、髪・ポニテ・服の一部が隣の900pxセルにはみ出すことがあります。

単純に

```text
900×900で25分割
```

すると、次の問題が出ます。

- 右上/左上を見るコマに、横のキャラが混入する
- 髪の上やポニテが見切れる
- 下段のキャラ頭部が上段コマに入り込む

そのため、現在は `tools/slice_character_sheets.py` の **component mode** を使います。

これはシート全体からキャラ単位の連結成分を検出して、25方向へ割り当てる方式です。

### 2. グレー背景の半透明ゴミに注意

元PNGが透過に見えても、境界付近に半透明のグレー背景が残ることがあります。

症状:

- キャラ周囲に薄い灰色の四角や線が見える
- 黒背景やOBSクロマキー時に境界が汚くなる

現在の生成スクリプトでは `--remove-gray-residue` を使って低彩度グレー残りを除去します。

### 3. 公開版の最終フレーム形式

公開版では、軽量化のため WebP 形式のスライス画像を参照します。

```text
characters/{キャラID}/slices/{A-F}/r{行}c{列}.webp
```

キャラ定義は `src/character-config.js` で管理します。

## 生成手順

例: `upamune` の新素材フォルダが `新キャラ資料/upamune` の場合。

```powershell
python tools\slice_character_sheets.py `
  --source "新キャラ資料/upamune" `
  --sheets-out "sheets/upamune" `
  --uploads-out "uploads/upamune" `
  --slices-out "public/characters/upamune/slices" `
  --format webp `
  --jobs 4 `
  --component-mode `
  --min-component-area 80 `
  --alpha-threshold 64 `
  --remove-gray-residue
```

例: `michiru_da` の場合。

```powershell
python tools\slice_character_sheets.py `
  --source "新キャラ資料/michiru_da" `
  --sheets-out "sheets/michiru_da" `
  --uploads-out "uploads/michiru_da" `
  --slices-out "public/characters/michiru_da/slices" `
  --format webp `
  --jobs 4 `
  --component-mode `
  --min-component-area 80 `
  --alpha-threshold 64 `
  --remove-gray-residue
```

生成後、各キャラの出力は以下になります。

```text
public/characters/{キャラID}/slices/A/r0c0.webp ... r4c4.webp
public/characters/{キャラID}/slices/B/r0c0.webp ... r4c4.webp
...
public/characters/{キャラID}/slices/F/r0c0.webp ... r4c4.webp
```

## 生成ログで見るポイント

各シートで以下のように出ていれば正常です。

```text
components=xxxx large=25
row0 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
row1 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
...
row4 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
```

確認ポイント:

- `large=25` になっている
- 各行の `comps(area)` が全部 `1:xxxxx`
  - 1セルに1キャラ成分だけ割り当たっている状態

NG例:

- `large=24` や `large=26`
- どこかが `0:0`
- どこかが `2:xxxxx`

この場合は切り抜き検出が失敗している可能性があります。

調整候補:

1. Best: `--alpha-threshold` を調整する
   - ゴミが多い: `80` や `96`
   - 細い髪が消える: `32` や `48`
2. Second-best: `--min-component-area` を調整する
   - 小物や髪束が別成分になる場合は小さめ
   - ノイズを拾う場合は大きめ
3. Not recommended: `--no-component-mode` で単純セル切り
   - 隣キャラ混入・髪見切れが再発しやすい

## 必ずやる検証

### 1. 枚数確認

```powershell
$characterId = 'upamune'
foreach ($s in 'A','B','C','D','E','F') {
  $webp = (Get-ChildItem -Path (Join-Path "public/characters/$characterId/slices" $s) -Filter '*.webp' | Measure-Object).Count
  Write-Output "$s webp=$webp"
}
```

全て `25` ならOKです。

### 2. 代表コマを目視確認

最低限見るコマ:

```text
public/characters/{キャラID}/slices/A/r0c0.webp  左上を見る
public/characters/{キャラID}/slices/A/r0c4.webp  右上を見る
public/characters/{キャラID}/slices/A/r2c2.webp  正面
public/characters/{キャラID}/slices/F/r0c0.webp  目閉じ口開け・左上
public/characters/{キャラID}/slices/F/r0c4.webp  目閉じ口開け・右上
public/characters/{キャラID}/slices/F/r2c2.webp  目閉じ口開け・正面
```

見るポイント:

- 横のキャラが混ざっていない
- 髪の上、ポニテ、耳飾りが切れていない
- 足元/服の下が変に切れていない
- 灰色の四角や背景残りがない
- A〜Fで位置がズレていない

### 3. 数値検査の基準

基準値:

- キャンバス: 1200×1200
- キャラ中心X: 約600px
- 足元Y: 900px
- A〜F状態間の外枠ズレ: 最大1px程度
- キャンバス端への接触: なし

## アプリ側の確認

Web版のキャラ切り替えは `src/character-config.js` で管理しています。

- 新しいキャラを追加する場合は `characters` 配列に `{ id, label, basePath, ext }` を追加する
- `basePath` は `public/` からの相対パスにする
- 既存キャラを差し替える場合は、同じ `basePath` にスライス画像を再生成する

## ローカル確認

```powershell
npm run dev
```

ブラウザで開く:

```text
http://localhost:5173/talk.html
http://localhost:5173/guruguru.html
```

確認ポイント:

- 画面右上または左上のキャラ選択で切り替えられる
- マウス追従で25方向が自然に切り替わる
- 左上/右上方向で横のキャラが出ない
- 口パク時にA/B/C、D/E/Fの位置が跳ねない
- まばたき時に顔位置がズレない
- 画像未配置のキャラは「画像準備中」と表示される

## 次回の推奨フロー

1. キャラIDを決める
2. 新キャラ6シートを `新キャラ資料/{キャラID}/` に入れる
3. `ffprobe` で4500×4500 RGBAか確認
4. `tools/slice_character_sheets.py` を component mode で実行し、`public/characters/{キャラID}/slices` に出力する
5. ログで `large=25` と各セル `1:xxxxx` を確認
6. A/Fの5×5全体を目視確認
7. 数値検査で中心X/足元Y/端接触を確認
8. `src/character-config.js` にキャラ定義があることを確認
9. `npm run dev` で実動作確認

この順番でやると、切り抜きミスやキャラ定義漏れを早い段階で発見できます。
