# 新キャラ差し替え手順・注意点

このメモは、キャラクター差し替え時に起きやすい「隣キャラの混入」「髪の見切れ」「透明境界の荒れ」を防ぐための手順です。

## 必要な素材

新キャラ用に、以下の6枚を用意します。

| シート | 目 | 口 | ファイル名の例 |
|---|---|---|---|
| A | 開け | とじ | `A_目開け_口とじ.png` |
| B | 開け | 中間 | `B_目開け_口中間.png` |
| C | 開け | 開け | `C_目開け_口開け.png` |
| D | 閉じ | とじ | `D_目閉じ_口とじ.png` |
| E | 閉じ | 中間 | `E_目閉じ_口中間.png` |
| F | 閉じ | 開け | `F_目閉じ_口開け.png` |

条件:

- 4500×4500px の透過PNG
- 5×5グリッド、1コマ900×900px
- 行: 上向き → 下向き
  - `r0`: 上を見る
  - `r1`: やや上
  - `r2`: 正面
  - `r3`: やや下
  - `r4`: 下を見る
- 列: 左向き → 右向き
  - `c0`: 左を見る
  - `c2`: 正面
  - `c4`: 右を見る
- A〜Fでキャラ位置・角度・体の大きさは同じにする
- 目と口以外はできるだけ同一にする

## 重要な注意点

### 1. 900pxセルで単純に切らない

素材によっては、髪・ポニテ・服の一部が隣の900pxセルにはみ出すことがあります。

単純に

```text
900×900で25分割
```

すると、次の問題が出ます。

- 右上/左上を見るコマに、横のキャラが混入する
- 髪の上やポニテが見切れる
- 下段のキャラ頭部が上段コマに入り込む

そのため、現在は `tools/slice_character_sheets.py` の **component mode** を使います。

これはシート全体からキャラ単位の連結成分を検出して、25方向へ割り当てる方式です。

### 2. グレー背景の半透明ゴミに注意

元PNGが透過に見えても、境界付近に半透明のグレー背景が残ることがあります。

症状:

- キャラ周囲に薄い灰色の四角や線が見える
- 黒背景やOBSクロマキー時に境界が汚くなる

現在の生成スクリプトでは `--remove-gray-residue` を使って低彩度グレー残りを除去します。

### 3. 公開版の最終フレーム形式

トマリ公開版では、軽量化のため現在 WebP 形式のスライス画像を参照しています。

```text
slices2/{A-F}/r{行}c{列}.webp
```

品質優先で PNG にする場合は、`app.jsx` と `talk-app.jsx` の参照拡張子も `.png` に変更します。

## 生成手順

例: 新素材フォルダが `新キャラ資料` の場合。

```powershell
python tools\slice_character_sheets.py `
  --source "新キャラ資料" `
  --sheets-out "sheets" `
  --uploads-out "uploads" `
  --slices-out "slices2" `
  --format webp `
  --jobs 4 `
  --component-mode `
  --min-component-area 80 `
  --alpha-threshold 64 `
  --remove-gray-residue
```

生成後、以下になります。

```text
slices2/A/r0c0.webp ... r4c4.webp
slices2/B/r0c0.webp ... r4c4.webp
...
slices2/F/r0c0.webp ... r4c4.webp
```

合計150枚です。

## 生成ログで見るポイント

各シートで以下のように出ていれば正常です。

```text
components=xxxx large=25
row0 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
row1 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
...
row4 comps(area) 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx 1:xxxxx
```

確認ポイント:

- `large=25` になっている
- 各行の `comps(area)` が全部 `1:xxxxx`
  - 1セルに1キャラ成分だけ割り当たっている状態

NG例:

- `large=24` や `large=26`
- どこかが `0:0`
- どこかが `2:xxxxx`

この場合は切り抜き検出が失敗している可能性があります。

調整候補:

1. Best: `--alpha-threshold` を調整する  
   - ゴミが多い: `80` や `96`
   - 細い髪が消える: `32` や `48`
2. Second-best: `--min-component-area` を調整する  
   - 小物や髪束が別成分になる場合は小さめ
   - ノイズを拾う場合は大きめ
3. Not recommended: `--no-component-mode` で単純セル切り  
   - 隣キャラ混入・髪見切れが再発しやすい

## 必ずやる検証

### 1. 枚数確認

```powershell
foreach ($s in 'A','B','C','D','E','F') {
  $webp = (Get-ChildItem -Path (Join-Path 'slices2' $s) -Filter '*.webp' | Measure-Object).Count
  Write-Output "$s webp=$webp"
}
```

全て `25` ならOKです。

### 2. 代表コマを目視確認

最低限見るコマ:

```text
slices2/A/r0c0.webp  左上を見る
slices2/A/r0c4.webp  右上を見る
slices2/A/r2c2.webp  正面
slices2/F/r0c0.webp  目閉じ口開け・左上
slices2/F/r0c4.webp  目閉じ口開け・右上
slices2/F/r2c2.webp  目閉じ口開け・正面
```

見るポイント:

- 横のキャラが混ざっていない
- 髪の上、ポニテ、耳飾りが切れていない
- 足元/服の下が変に切れていない
- 灰色の四角や背景残りがない
- A〜Fで位置がズレていない

### 3. 数値検査の基準

基準値:

- キャンバス: 1200×1200
- キャラ中心X: 約600px
- 足元Y: 900px
- A〜F状態間の外枠ズレ: 最大1px程度
- キャンバス端への接触: なし

## アプリ側の確認

現在の参照先:

- `app.jsx`
  - `slices2/A/r{r}c{c}.webp`
  - まばたき時 `slices2/D/r{r}c{c}.webp`
- `talk-app.jsx`
  - `slices2/{sheet}/r{r}c{c}.webp`

PNG以外に変更する場合は、ここも合わせて変更します。

## ローカル確認

```powershell
python -m http.server 8000
```

ブラウザで開く:

```text
http://localhost:8000/トマリトーク.html
```

確認ポイント:

- マウス追従で25方向が自然に切り替わる
- 左上/右上方向で横のキャラが出ない
- 口パク時にA/B/C、D/E/Fの位置が跳ねない
- まばたき時に顔位置がズレない

## 次回の推奨フロー

1. 新キャラ6シートを専用フォルダに入れる
2. `ffprobe` で4500×4500 RGBAか確認
3. `tools/slice_character_sheets.py` を component mode で実行
4. ログで `large=25` と各セル `1:xxxxx` を確認
5. A/Fの5×5全体を目視確認
6. 数値検査で中心X/足元Y/端接触を確認
7. ローカルサーバーで実動作確認

この順番でやると、今回のような切り抜きミスを早い段階で発見できます。
